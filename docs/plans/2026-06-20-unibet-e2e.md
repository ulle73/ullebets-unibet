# Ullebets Unibet E2E Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build and verify the missing `ullebets-unibet` pipeline so it reads source matches from `app.teamstats`, writes normalized matches and fetch jobs into `ullebets_unibet`, and stores full raw Kambi payload snapshots with traceable metadata.

**Architecture:** Keep `MONGODB_DB` reserved for the target database `ullebets_unibet` and read source data explicitly from `app`. Split the new pipeline into small modules for source extraction, match scheduling, Kambi discovery/fetch, and reporting so each behavior can be tested independently and the CLI scripts remain thin.

**Tech Stack:** Node.js ESM, built-in `node:test`, MongoDB driver, dotenv, Kambi/Unibet HTTP fetch
